## Key Columns

### `bom`
```sql
bom_id text UNIQUE          -- YYMMDD-N-MODEL (generated by DB function)
date date                   -- ngày tính giá
product_type text           -- loại sản phẩm
so_mo text                  -- số SO/MO
model text                  -- tên model
price_list_type text        -- loại bảng giá
sp_type text                -- Basic | Fancy
cost_gold/stones/labor/subtotal/cif/total numeric
sell_price numeric
discount_pct numeric        -- 0.05 = 5%
discount_price numeric      -- giá sau chiết khấu
img1/img2/img3 text         -- Google Drive URL hoặc fileId
folder_url text             -- Google Drive folder URL
created_by text             -- username
sales_person text
store text                  -- VN | US | ADM
```

### `gold_material`
```sql
price_date date UNIQUE
amark_gold_oz numeric       -- giá vàng/oz từ Amark
amark_pt_oz numeric         -- giá bạch kim/oz
amark_ag_oz numeric         -- giá bạc/oz
loss_factor numeric         -- default 1.06
karat_prices jsonb          -- {"10K": 21.5, "18K": 38.8, "PT": 32.5, ...}
```

### `stone_material`
```sql
group_code text             -- mã nhóm đá (= dm_size.master_code)
grade_id text UNIQUE        -- mã grade cụ thể
type_input text             -- 'mm' | 'ct'
min_size / max_size numeric -- range lookup
selling_price numeric       -- = base_price × (1 + mkup)
unit text                   -- pricing unit ('ct' | 'pc')
```

### `mk_store_markup`
```sql
value_from / value_to numeric   -- cost_total range
markups jsonb                   -- {"1)HPUS -P": 1.8, "B1)HPVN -P": 2.1, ...}
```

### `sys_config`
```sql
key   -- 'VND_RATE', 'PROCESS_FEE_LAPRAP'
value -- string value
```

### `audit_log`
```sql
actor text           -- username
role text
action text          -- CREATE|UPDATE|DELETE|DISCOUNT
entity text          -- bom|gold|user|stone|mk|config
entity_id text
summary text
diff jsonb           -- {before: {...}, after: {...}}
created_at timestamptz
```

## PostgreSQL Functions

| Function | Params | Mô tả |
|---|---|---|
| `generate_bom_id(date, model)` | `p_date date, p_model text` | Tạo BOM ID: YYMMDD-N-MODEL |
| `lookup_stone_grade(group_code, mm_size, ctw1pc)` | 3 params | Lookup stone_material theo range |
| `sync_dm_size_to_stone_material()` | none | Sync dm_size → stone_material (xóa + insert) |
| `auth_role()` | none | Lấy role từ JWT claims |
| `auth_store()` | none | Lấy store từ JWT claims |
| `set_updated_at()` | trigger | Auto-update updated_at |

## RLS Rules

- **bom**: Admin/Manager xem tất cả. User thường xem BOM của mình + cùng store
- **users**: chỉ Admin
- **sys_config**: ai cũng đọc, Admin/Manager ghi
- **gold_material, stone_material, dm_size**: ai cũng đọc, Admin ghi

> **Lưu ý**: API routes dùng `createServiceClient()` để bypass RLS — queries tự do. Dùng `createClient()` chỉ để verify session.
